# Detection and tf2 localization

## Purpose
Detect simulated victims and publish their estimated positions in the global `map` frame.

## MVP strategy
Start with simple colored victims or ArUco/AprilTag markers. Avoid heavy human detection models such as YOLO unless the complete project already works.

## Inputs
- `/camera/image_raw` — camera image from Gazebo.
- `/camera/camera_info` — camera calibration information.
- `/tf` and `/tf_static` — transformations between camera, robot, odom and map frames.

## Outputs
- `/victims_map` — `geometry_msgs/msg/PoseArray` containing victim positions in `map`.
- `/visualization_marker_array` — RViz markers generated by the results/visualization module.

## Required transform
The detector/localizer must transform detections from the camera frame to the map frame:

```text
camera_link or camera_optical_frame -> base_link -> odom -> map
```

## First implementation target
1. Subscribe to the image topic.
2. Detect a simple visual target.
3. Estimate a relative pose.
4. Transform the pose into `map` with `tf2_ros`.
5. Publish `/victims_map`.
